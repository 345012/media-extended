# 已知 Bug

## YouTube 视频在网页播放器中自动暂停 [#youtube-video-auto-pause-issue]

[问题 #219](https://github.com/PKM-er/media-extended/issues/219)

用户反馈了一个问题：在网页播放器中播放 YouTube 视频时，视频会在开始播放后只播放了一帧就自动暂停。尽管尝试了多种方法来解决这个问题，包括安装不同的版本和采用推荐的修复措施，但问题仍间歇出现，某些版本受影响，而其他版本则不受影响。

### 解决方案

针对这个问题，v3.0.5 及以上版本提出了一个潜在的解决方法，需要通过命令设置一个特定的Flag：

```text copy
DEV: Disable Partition
```

使用这个命令后，用户需要重启应用程序来使修复生效。其通过关闭一个可能导致自动暂停的特定功能来解决这个问题。这个解决方案的效果可能会有所不同，需要进一步的测试来验证其可靠性。

因此，我们强烈建议采取这个临时解决方案的用户分享其实际效果，包括任何改善或仍然存在的问题。为了促进更详细和有益的反馈，请通过执行 Obsidian 的 `show debug info` 命令来提供完整的环境信息，并在[ GitHub 上的原问题](https://github.com/PKM-er/media-extended/issues/219#top)中报告。

<Callout type="info">
  YouTube 视频自动暂停的问题可能与 Electron 框架底层的处理有关，尤其是关于如何管理 `webview` 标签的 `partition` 属性的问题。由于问题的复杂性，可能难以找到一个彻底的解决办法。
</Callout>
